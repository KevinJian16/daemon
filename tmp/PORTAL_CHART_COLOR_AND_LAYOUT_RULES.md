# Portal 图表配色与构图规则

## 1. 目的

这份规则只约束 `Portal` 里的图表型视图：

- `Slip` 的 `Move` 结构图
- `Deed` 的 `Move` 运行图
- `Folio` 的卷内视图

它不负责整站品牌色，只负责回答三件事：

1. 图表该怎么配色
2. 图表里的元素该怎么画
3. 图表里的元素该怎么排布

---

## 2. 配色规则

### 2.1 总原则

- 页面背景必须很浅
- 图面背景也必须很浅
- 元素本体必须明显更深
- 强调态只允许一支强调色
- 不允许出现大量“差不多但不一样”的中间色

一句话：

`浅底 + 深元素 + 单一强调`

### 2.2 色阶分工

图表只允许四类颜色角色：

1. 页面底
2. 图面底
3. 元素色
4. 强调色

约束：

- 页面底和图面底最多使用两档浅色
- 元素色最多使用两档深色
- 强调色最多使用一档
- 不允许再插入第三类“中间暖灰 / 中间褐灰 / 中间脏米色”

### 2.3 中性色规则

- 中性色只用来表达层级，不用来“调感觉”
- 中性色必须来自稳定色阶，而不是现场手拌
- 文本、边框、线条、弱背景必须各自占固定档位

结论：

- 以后只从成熟色阶里取色
- 不再临时调一堆很接近的自定义米灰

### 2.4 强调色规则

- `Portal` 当前主动作色可以作为图表强调色来源
- 但强调色不能占满整张图
- 强调色只用于：
  - 当前运行态
  - 当前激活态
  - 当前最重要动作

不用于：

- 所有节点一起高亮
- 所有边一起染色
- 所有状态点一起抢主色

### 2.5 当前执行约束

针对当前这套 `Portal`：

- `Slip Move`：
  - 背景浅
  - 节点与线条为深中性
  - 不使用大面积强调色
- `Deed Move`：
  - 复用 `Slip Move` 骨架
  - 只把当前流动层改成强调色
- `Folio`：
  - 背景浅
  - `Slip` 卡片为深色对象
  - 关系线比对象轻，但必须清晰可见

---

## 3. 元素规则

### 3.1 `Slip Move`

`Slip Move` 是结构图，不是运行图。

因此：

- 节点是稳定骨架
- 节点标题短
- 节点不承载长摘要
- 线条只表达结构连接
- 不加运行态动势

视觉上应该先让用户读出：

- 有几步
- 先后关系
- 分叉与汇合

### 3.2 `Deed Move`

`Deed` 不是另一张图，而是同一张 `Slip Move` 的时间态。

因此：

- 骨架与 `Slip Move` 保持一致
- 只额外叠加：
  - 当前活跃节点
  - 已走过节点
  - 当前流动边
- 差异主要靠活性层，不靠重做一张新图

### 3.3 `Folio`

`Folio` 不是 `DAG` 图。

它是卷内对象空间。

因此：

- `Slip` 在 `Folio` 里是对象卡，不是步骤节点
- 关系线是对象间关系，不是流程线
- 时钟位、状态点、进入详情，都是对象卡上的局部信号
- `Folio` 图必须看起来像“卷内摆放”，不能像“流程图”

### 3.4 文本规则

- 节点标题必须短
- 节点里不放解释段落
- 图里的文字只保留识别对象所需内容
- 长说明放图外，不塞入节点

### 3.5 装饰规则

- 不使用过多纹理
- 不使用重阴影堆层次
- 不使用多种边框色
- 不使用多种近似背景色

图表的层次优先靠：

1. 位置
2. 对比
3. 线型
4. 状态

不是靠很多装饰。

---

## 4. 排布规则

### 4.1 网格与对齐

- 节点必须挂在稳定网格上
- 分叉和汇合必须清楚
- 节点之间保留足够呼吸距离
- 连线弯折要稳定，不要每条都不一样

### 4.2 缩略图与全图

- 页内默认放缩略图
- 缩略图只承担“看结构”
- 放不全时，不强行压缩到难读
- 通过放大镜进入全图
- 全图是同一对象的展开态

### 4.3 点击与退出

- 缩略图和全图的交互模式必须一致
- 全图一律支持：
  - `Esc`
  - 点遮罩空白处退出
- 不再额外发明返回按钮

---

## 5. 以后改图时的检查清单

每次改图表前先问：

1. 这张图到底是结构图、运行图，还是对象空间图？
2. 当前是否用了超过两档浅底？
3. 当前是否用了超过两档元素色？
4. 当前是否塞进了中间脏色？
5. 当前强调色是不是只留给活跃态？
6. 当前节点里是不是放了太多字？
7. 当前缩略图是不是已经难以辨认？

只要其中两条答错，就先回退，不继续加细节。

这份规则主要参考以下成熟方案和官方文档，并结合 `Portal` 的对象语义做了归纳：

- [Atlassian Data Visualization Color](https://atlassian.design/foundations/color-new/data-visualization-color/)

归纳结论：

- UI 配色和图表配色要分角色管理
- 图表里最重要的是层级和强调分工，不是“找一组顺眼颜色”
- 流程图、对象图、运行图不能只靠换色区分，更要靠元素语义和排布区分
