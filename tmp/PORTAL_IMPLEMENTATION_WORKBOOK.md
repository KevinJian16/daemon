# Portal 实施工作簿

> 作用：这是一份实现工作簿，不是权威机制文档。  
> 这份文档只记录当前已经对齐的前台实现规则，用来约束实现、审视页面、回溯设计。  
> 权威机制仍以 `.ref/` 下正式文档为准。

## 1. 总原则

### 1.1 Claude 壳原则

- 能直接拿 `Claude` 的地方，直接拿。
- `Portal` 的整体页面布局、左栏、主区、输入区、搜索入口、条目样式、条目右侧 `…`、收起方式、密度、留白、主次层级，都直接对齐 `Claude`。
- 左栏的显示机制、折叠方式、条数限制、层级节奏也直接对齐 `Claude`。

### 1.2 Apple 动态原则

- 能直接拿 `Apple` 的地方，直接拿。
- 所有会动的东西都按 `Apple` 的逻辑和手感实现。
- 选择、拖动、整理模式、比较切换、危险动作确认、对象收束、系统先帮用户摆好再完全听用户的规则，都直接对齐 `Apple`。

### 1.3 只设计对象内容

- 真正需要自己设计的，只有系统自己的对象内容。
- Portal 自己定义的内容只有：
  - `Folio` 页里装什么
  - `Slip` 页里装什么
  - `Deed` 在 `Slip` 里怎么呈现
  - 时钟、关系线、比较、收束这些对象内容

### 1.4 恶意行为原则

- 所有前台动作、后台结构、收束窗口、删除语义、条数限制，都要考虑恶意行为。
- 需要防住：
  - 恶意创建空卷
  - 恶意不成札地长聊
  - 恶意复制、拖动、连线、加时钟
  - 恶意不收束 `Deed`
  - 恶意堆积候选版本
  - 恶意制造无穷规则或垃圾对象

## 2. Portal 正式对象

- 前台正式对象：
  - `Draft`
  - `Slip`
  - `Folio`
  - `Deed`
- `Writ` 是后台规则对象，在前台通过线、时钟、调度结果被看见。
- 消息流是承载层，不是导航对象。

### 2.1 视图层映射

- `Folio` 在视图层等于 `Claude` 的 `Project`
- `Slip` 是系统自己的结构对象
- `Deed` 是 `Slip` 上的一次行为
- `Draft` 是尚未成札的对象

## 3. Portal 整体布局

- `Portal` 的整体壳直接对齐 `Claude`
- 主区是单主列
- 对话流为主
- `Draft / Slip / Folio / Deed` 的内容卡嵌在主列中
- 输入区固定在底部
- 静态壳用 `Claude`
- 动态语法用 `Apple`
- 对象操作采用文件系统式直觉

## 4. 左栏

### 4.1 左栏正式分区

左栏固定三块：
1. `案头`
2. `卷`
3. `散札`

### 4.2 左栏机制

- 左栏结构、条目密度、hover 行为、显示条数、折叠方式全部直接对齐 `Claude`
- 左栏不自己发明树状结构
- 左栏不自己发明复杂预览

### 4.3 案头

案头承接：
- 尚未成札、但已具事项性的线程
- 已形成 `Draft`、但尚未成札的线程

案头规则：
- 理想显示 `3` 项
- 硬上限 `5` 项
- `24h` 未活动降权
- `72h` 未活动退出案头
- 不支持手动钉住
- 成札后立即退出案头

### 4.4 卷

- `Folio` 显示在左栏卷区
- 卷项样式直接对齐 `Claude project` 条目
- 卷项不在左栏里展开内部内容

### 4.5 散札

- `散札` 是卷外正式 `Slip`
- 左栏中的显示机制和条数限制直接对齐 `Claude`
- 更多散札通过搜索和归档找回

## 5. 搜索

- 搜索入口固定在左栏顶部
- 支持 `cmd/ctrl + k`
- 搜索面板样式和行为对齐 `Claude`

### 5.1 搜索命中对象

搜索命中：
1. `卷`
2. `签札`

未成札线程不进入长期搜索对象世界。

### 5.2 搜索结果

- 结果在同一面板内显示
- 使用轻分组
- 不用 tab
- 分组顺序固定为：
  1. `卷`
  2. `签札`
- 排序规则：
  - 相关性优先
  - 时间次之

### 5.3 空搜索

- 空搜索显示少量“继续上次”对象
- 条数和展示方式直接对齐 `Claude`
- 不显示全历史

### 5.4 跳转

- 点 `卷` -> 打开该 `Folio` 页
- 点 `签札` -> 打开该 `Slip` 页

## 6. 删除与移除

Portal 的正式动作固定为三种：
1. 从卷中取出
2. 归档
3. 彻底删除

### 6.1 从卷中取出

- `Slip` 仍然存在
- 不再属于当前 `Folio`
- 变成卷外 `Slip`

### 6.2 归档

- 对象仍然存在
- 保留历史
- 离开当前工作视野

适用于：
- `Slip`
- `Folio`

### 6.3 彻底删除

适用于：
- `Slip`
- `Folio`

`Slip` 彻底删除时：
- 删除 `Slip`
- 删除关系
- 删除节律
- 删除相关 `Writ`
- 删除对外对象层的可见结构
- 向知识系统发送来源状态更新

`Folio` 彻底删除时：
- 空卷可直接删除
- 非空卷必须提供两个明确选项：
  1. 解卷并保留内部签札
  2. 连内部签札一起彻底删除
- 默认高亮：
  - 解卷并保留内部签札

### 6.4 批量删除

- 允许批量彻底删除
- 必须强确认

### 6.5 局部删除

- 删除线：删除关系
- 删除时钟：删除节律

## 7. 草稿

### 7.1 位置

- `Draft` 只出现在当前主区线程里
- `Draft` 不进入左栏一级项
- `Draft` 不进入长期搜索对象结果
- 左栏 `案头` 只对草稿做轻提示

### 7.2 形式

`Draft` 显示为一张明确的草稿卡，固定显示：
1. 标题
2. 一句目标摘要
3. 当前收敛状态
4. 当前方案的极简摘要
5. 明确的“尚未成札”提示

### 7.3 内容状态

- 一张草稿只有一个当前对象
- 多方案属于这张草稿内部的内容状态
- 前台不并排显示多个草稿对象

### 7.4 成札后的保留内容

`Draft -> Slip` 后，只保留：
1. 当前方案
2. 既往方案设计

### 7.5 视觉与动画

- 草稿卡和正式签札卡属于同一组件族
- 草稿卡更轻、更未定稿
- 正式签札卡更稳定、更收束
- 成札时使用 `Apple / macOS` 式对象收束动画

## 8. Slip 页

### 8.1 定位

`Slip` 页是 Portal 主战场。

### 8.2 固定结构

`Slip` 页固定包含：
1. 签札题头
2. 当前 `Slip` 结构卡
3. 历次 `Deed` 列表
4. `Slip` 对话流

点击任一 `Deed` 后，进入该 `Deed` 详情页。  
详情页必须提供明确返回按钮，返回当前 `Slip`。

### 8.2.1 对象骨架

- `Slip` 的主视图不是说明页，而是一张 `DAG`
- `move` 是 `DAG` 上的节点，不是普通列表项
- `Slip` 首屏必须先看到这张图，再看到解释性信息
- `Slip` 页只保留 `Slip` 自己的状态、按钮、节律和对话
- 历次 `Deed` 只以列表出现，不在 `Slip` 页里展开成第二套对象
- 不允许把 `Slip` 重新做成后台式信息栅格
- `Move` 图如果在页内放不全，必须和 `Folio` 一样使用“缩略图 + 全图展开态”
- `Move` 图与 `Folio` 图的配色和构图，统一遵守：
  - [PORTAL_CHART_COLOR_AND_LAYOUT_RULES.md](./PORTAL_CHART_COLOR_AND_LAYOUT_RULES.md)

### 8.2.2 前台信息约束

- `Slip` 页只显示用户判断当前对象所需的信息
- 不显示内部技术标识：
  - `deed_id`
  - `slip_id`
  - 内部 route 名称
  - 内部状态键名
- 前台优先显示：
  - 这张札在说什么
  - 当前处在哪个阶段
  - 现在该做什么
  - 最近这轮产出了什么

### 8.3 主动作

`Slip` 页提供明确主动作：
- 开始执行

对话里说“开始”“执行这张札”，也走同一个动作。

### 8.4 对话框行为

对话框可以：
- 收敛草稿
- 成札
- 组织卷
- 调整当前未 `closed` 的 `Deed`
- 启动新的 `Deed`

### 8.4.1 统一输入口

- `Portal` 在 `Draft / Slip / Folio` 页面里使用**同一个物理输入口**
- 输入口的位置、外观、尺度、开合节奏保持一致
- 不因当前 owner 是 `Draft / Slip / Deed / Folio` 而更换另一种输入框组件
- 输入口必须显式显示当前 owner
- owner 变化时，要有明确可感知的状态切换提示，不能静默换语义

### 8.4.2 owner 切换

同一个输入口的 owner 只允许在以下对象间切换：

1. `Draft`
2. `Slip`
3. 当前未 `closed` 的 `Deed`
4. `Folio`

规则：

- `Draft` 页：输入口 owner = 当前 `Draft`
- `Slip` 页且无活跃 `Deed`：输入口 owner = 当前 `Slip`
- `Slip` 页且最近一次 `Deed` 为 `running / settling`：输入口 owner = 当前 `Deed`
- `Slip` 页且最近一次 `Deed` 为 `closed`：输入口 owner 回到当前 `Slip`
- `Folio` 页：输入口 owner = 当前 `Folio`

### 8.4.3 语义控制与结构控制

- 语义控制优先通过输入口表达：
  - 改目标
  - 改当前方案
  - 追加要求
  - 请求再来一遍
- 结构控制优先通过直接操作表达：
  - 收入卷
  - 从卷中取出
  - 重排
  - 连线
  - 设置时钟

因此：

- 输入口 owner 可以变
- 输入口组件不能乱变
- 不是所有动作都应该塞进输入口
- `settling` 中的比较、选版本、评价是阶段页主交互，不是主要依赖聊天完成

### 8.5 文件上传

- 文件上传默认进入当前 `Draft` 或 `Slip` 的材料集
- `Deed` 启动时，对材料集做快照
- 本次 `Deed` 使用启动时快照

## 9. Folio 页

### 9.1 定位

`Folio` 在视图层等于 `Claude project`。  
`Folio` 页使用 `Claude project` 页面骨架。

### 9.2 固定结构

`Folio` 页固定包含：
1. 卷题头
2. 小型卷内视图卡
3. 卷内对话流
4. 整理动作区（仅整理态可见）

`Folio` 的“全图”不是另一套页面，而是卷内视图卡的展开态。  
展开后必须提供明确返回按钮，返回 `Folio` 主页。

### 9.3 卷内视图板

- `Folio` 主页默认显示缩略卷内视图
- 缩略视图中的每张 `Slip` 只显示：
  - 名称
  - 一个颜色点表示当前状态
  - 一个时钟标识表示节律开关
- 点击“查看详情”后，展开成全屏卷内视图
- 不管在缩略图还是全图里，点击 `Slip` 都直接进入 `Slip` 详情页
- 从 `Slip` 详情页必须能明确返回到刚才所在的 `Folio`
- 默认清楚展示 `3` 张签札
- 常态清楚展示 `5` 张签札
- 超过 `5` 张时，扩展成可滚动视图板，不缩成难以辨认的小卡片
- 当前焦点签札最突出
- 其余签札退后但保持可见
- 视图板必须先于解释性列表出现
- 关系线、时钟位、焦点切换都直接在视图板上完成
- 卷内视图板的配色和构图，统一遵守：
  - [PORTAL_CHART_COLOR_AND_LAYOUT_RULES.md](./PORTAL_CHART_COLOR_AND_LAYOUT_RULES.md)

### 9.4 卷内签札卡

卷中的每张 `Slip` 显示为卷内摘要卡，固定显示：
1. 标题
2. 一句目标摘要
3. 当前状态
4. 最新进展一句
5. 结果提示

如果该签札带节律：
- 额外显示时钟标记
- 点时钟后再展开节律细节

### 9.5 阅读模式与整理模式

卷页有两个模式：
- 阅读模式
- 整理模式

整理模式是结构编辑模式，也是生效开关。
整理模式必须持续存在，直到用户显式退出；做一次移动、连线、取出卷、设置时钟，不得自动退回阅读模式。

### 9.6 整理模式支持的动作

- 选中
- 拖动
- 连线
- 设置时钟
- copy
- 归档
- 彻底删除
- 收入卷
- 从卷中取出
- 开始执行当前签札

退出整理模式后：
- 当前结构和规则已经正式生效
- 不出现额外保存确认

### 9.7 选择

`selection` 使用 Apple / 文件系统式多选逻辑。

桌面端：
- `cmd/ctrl + 点击`
- `shift + 点击`
- 点击空白取消

移动端：
- 长按进入选择态

批量操作支持：
- 删除 / 归档
- 执行
- 拖动
- copy
- 收入卷
- 从卷中取出

### 9.8 关系与节律

- 线表示：前一札完成后触发后一札
- 前台触发图保持 DAG
- 允许：
  - 一对多
  - 多对一
- 多对一语义固定为：
  - 所有前置条件满足后启动
- 时钟直接长在签札卡上
- 点空的时钟位，给该札添加节律
- 点已有时钟，查看和修改节律
- 删除时钟，移除节律
- 节律设置面板使用 Apple 式设置方式

### 9.9 重复执行

- 图中不放环
- 图中不放自回环
- 定时重复靠时钟
- 手动重复靠“重新运行”按钮

### 9.10 复制

`Slip` 支持 copy。

复制内容：
- 标题
- 目标
- brief
- design
- 节律配置

不复制：
- `slip_id`
- slug
- 历史 `Deed`
- 反馈
- 学习记录

复制结果默认出现在原札旁边。

如果原札带时钟：
- 复制后保留节律配置
- 时钟默认不激活
- 用户点一次时钟确认后才正式生效

## 10. 排序

### 10.1 默认行为

系统先自动把卷里的签札摆好。

默认摆法：
- 当前最重要的签札在最前
- 更相关的签札排在它附近
- 已收束的签札自动退后

### 10.2 用户重排

用户一旦手动拖动过签札，卷内顺序就以用户顺序为准。

后续系统：
- 不再改卷里的主顺序
- 只做轻量提示：
  - 高亮当前焦点
  - 提醒即将到点
  - 显示正在执行
  - 已收束签札退后

### 10.3 用户规则

用户只需要理解一句话：

**系统会先帮你摆好；你自己摆过之后，它就听你的。**

## 11. Deed、结果、比较、收束

### 11.1 基本关系

- 每个 `Slip` 都可以产生多个 `Deed`
- `Deed` 是一轮行事
- 不同 `Deed` 是不同轮次
- 候选版本只存在于单个 `Deed` 内部

### 11.2 Portal 里的显示

- `Slip` 页显示 `Deed` 列表
- 当前 `Deed` 和过往 `Deed` 都保留查看入口
- `Deed` 列表承担：
  - 告知每轮 `Deed` 当前或最终状态
  - 展示 `closed deed` 的产物标签
  - 进入该轮 `Deed` 页面

### 11.3 `Deed` 状态

一个 `Deed` 有三个关键状态：
1. `running`
2. `settling`
3. `closed`

`Deed` 不是另一张后台详情页，而是当前 `Slip DAG` 的时间态。
因此：

- `running` 时，进度在 `Slip DAG` 上流动
- `settling` 时，仍然保留同一张 `DAG`，只是主交互变成比较和选保留
- `closed` 时，仍然保留同一张 `DAG`，只是活跃层收束回正式结果
- `Deed` 展开页是同一对象的展开态，不是另一套后台骨架
- `Deed` 的状态、按钮、对话框都与 `Slip` 分离
- `Slip` 对话永远属于 `Slip`
- `Deed` 对话永远属于当前被打开的 `Deed`
- `Deed running` 的图层设计，统一遵守：
  - [PORTAL_CHART_COLOR_AND_LAYOUT_RULES.md](./PORTAL_CHART_COLOR_AND_LAYOUT_RULES.md)

### 11.3.1 `closed deed`

- `closed` 后只保留：
  - 产物标签
  - 对话历史
  - 查看入口
- `closed deed` 不再提供其它操作
- 过往 `Deed` 页面仍然保留
- 但需要有保质期，过期后在列表中逐渐淡出

含义：
- `running`
  - 正在执行
  - 页面只显示进行中信息
  - 不提供结果比较与版本选择
- `settling`
  - 本次 `Deed` 尚未结束
  - 可以比较版本
  - 可以改选版本
  - 可以评价
  - 可以改评价
  - 可以重新运行，产生新候选版本
- `closed`
  - 本次 `Deed` 正式结束
  - 版本冻结
  - 评价冻结
  - 再运行将生成新的 `Deed`

### 11.4 候选版本

- 候选版本只存在于 `running / settling`
- `closed` 后没有候选版本
- 默认情况下，一个 `Deed` 只产出一个结果
- 当同一个 `Deed` 在未结束阶段产生第二个及以上可比较结果时，该 `Deed` 进入候选版本状态
- 候选版本上限固定为 `5`
- 新生成的候选会挤掉最旧的候选

### 11.5 当前阶段页面

- `running` 和 `settling` 各有一个当前阶段页面
- `Deed` 卡点进去，打开当前阶段页面
- `closed` 后没有阶段页面
- 阶段结束，页面立即消失

### 11.6 比较

- `Slip` 详情页在最近一次 `Deed` 进入 `settling` 后提供“比较”入口
- 比较是当前阶段页面里的一个模式
- 进入比较后，先勾选要比较的版本
- 然后按同类产物逐页比较
- 每个同类产物占一页
- 用户在每一页里决定保留哪个版本
- 比较页是产品化比较页，不是 Git 工具页
- 系统内不做跨 `Deed` 比较

### 11.7 正式结果

- `Deed` 在 `settling` 结束前，正式结果尚未冻结
- 用户在比较后确定当前保留版本
- 被保留版本进入正式 `Offering`
- 未被保留的候选结果留在本次 `Deed` 的候选区，直到本次 `Deed` `closed` 后清理

### 11.8 评价

- 评价属于 `settling` 的一部分
- 评价可以和版本选择一起反复修改
- 评价始终绑定当前被选中的版本
- 改选版本后，评价需要重新确认
- `closed` 后，评价冻结

### 11.9 收束窗口

- `Deed` 执行结束后进入 `settling`
- `settling` 有时间窗口
- 普通任务：30 分钟
- 定时任务：`min(30 分钟, 周期)`

### 11.10 托管

- 产物托管开启时，默认保留最新版本
- 评价托管开启时，默认状态为未评价
- 超时后落默认行为
- `settling` 结束后，`Deed` 进入 `closed`

### 11.11 调整与重新运行

- `调整` 只通过对话框进行
- `重新运行` 是明确可点击的动作
- 在 `running / settling` 中重新运行，只给当前 `Deed` 增加新候选版本
- 在 `closed` 后重新运行，会新开一个 `Deed`

### 11.12 下游使用

- 下游使用的是上游 `Slip` 当前正式结果引用
- 启动下游 `Deed` 时，系统生成 `input_manifest`
- 是否使用了上游结果，以 `input_manifest` 为准
- 依赖上游结果的下游 `Slip`，只有在所有上游 `Deed` 已经 `closed` 后才能启动

### 11.13 约束

- 一张 `Slip` 同时只允许一个未 `closed` 的 `Deed`
- 候选版本只存在于未结束 `Deed`
- `closed` 后，这次 `Deed` 不再接受新版本

## 12. 状态机

### 12.1 `Draft`

状态：
1. `drafting`
2. `gone`

`drafting` 可以：
- 继续对话收敛目标
- 上传文件进入草稿材料集
- 改当前方案
- 成札
- 放弃

### 12.2 `Slip`

状态：
1. `active`
2. `archived`
3. `deleted`

`active` 可以：
- 打开详情页
- 开始新的 `Deed`
- 设置时钟
- 删除时钟
- 建立关系线
- 删除关系线
- copy
- 收入卷
- 从卷中取出
- 归档
- 彻底删除

同理，`Slip` 页的整理态也必须持续存在，直到用户显式退出；移动一步、插入一步，不得自动退出整理态。

### 12.3 `Folio`

状态：
1. `active`
2. `archived`
3. `deleted`

`active` 可以：
- 打开卷页
- 重命名
- 进入整理模式
- 接收 `Slip`
- 放出 `Slip`
- 调整卷内顺序
- 归档
- 彻底删除

### 12.4 `Deed`

状态：
1. `running`
2. `settling`
3. `closed`

`running` 可以：
- 查看进行中信息
- 在对话框里补充要求
- 停止

`settling` 可以：
- 比较
- 选版本
- 评价
- 改评价
- 重新运行
- 确认收束

`closed` 可以：
- 从 `Slip` 页查看最近一次 `Deed` 卡
- 打开最新正式 `Offering`
- 再次执行生成新的 `Deed`

### 12.5 输入口 owner 速查

- `Draft drafting` -> owner = `Draft`
- `Slip active` 且无活跃 `Deed` -> owner = `Slip`
- `Deed running` -> owner = 当前 `Deed`
- `Deed settling` -> owner = 当前 `Deed`
- `Deed closed` -> owner = `Slip`
- `Folio active` -> owner = `Folio`

## 13. 清理矩阵

### 13.1 `Draft: drafting -> gone`

- 当前草稿卡：删除
- 当前方案：成札时写入 `Slip`；放弃时删除
- 既往方案设计：成札时压缩写入 `Slip`；放弃时删除
- 草稿材料集：成札时并入 `Slip`；放弃时删除
- 普通噪声对话：删除

### 13.2 `Deed: running -> settling`

- `running` 页面：删除
- 当前目标快照：写入本次 `Deed` 记录
- 输入快照摘要：写入本次 `Deed` 记录
- `input_manifest`：保留为本次 `Deed` 正式记录
- 进行中日志流：删除
- 进行中摘要：压缩后写入本次 `Deed` 记录
- 本轮结果：进入当前 `Deed` 候选结果区
- 未生效的临时对话输入：删除
- 已生效的调整：写入本次 `Deed` 目标快照

### 13.3 `Deed: settling`

- 候选版本：最多保留 `5` 个
- 比较选择状态：只保留在当前页面内
- 当前被选中的版本：挂在本次 `Deed` 上
- 评价草稿：只保留当前版，改选后回到待确认
- 页面模式状态：只保留在当前页面内

### 13.4 `Deed: settling -> closed`

- `settling` 页面：删除
- 被选中的正式结果：进入 `Offering`
- 未被选中的候选结果：从活跃候选区清理
- 最终评价：写入本次 `Deed` 反馈记录
- 未确认或被覆盖的评价草稿：删除
- 比较临时状态：删除
- 最终目标快照：保留在本次 `Deed` 记录
- 最终 `input_manifest`：保留在本次 `Deed` 记录
- 下游正式结果引用：更新
- 向知识系统发送来源状态更新

### 13.5 `Slip: active -> archived`

- `Slip` 对象：保留
- `Slip` 详情页临时 UI 状态：删除
- 正式关系、时钟、材料集、历史 `Deed` 记录：保留
- 左栏活跃显示：移出

### 13.6 `Slip: active/archived -> deleted`

- `Slip` 对象：删除
- `Slip` 关系线：删除
- `Slip` 时钟：删除
- `Slip` 材料集：删除
- 当前阶段页面：删除
- 向知识系统发送来源状态更新

### 13.7 `Folio: active -> archived`

- `Folio` 对象：保留
- `Folio` 页临时 UI 状态：删除
- 卷内结构、时钟、关系、内部 `Slip`：保留
- 左栏活跃显示：移出

### 13.8 `Folio: active/archived -> deleted`

- 空卷：直接删除 `Folio`
- 非空卷：
  - 解卷并保留内部签札
  - 或连内部签札一起彻底删除
- 卷页临时 UI 状态：删除
- 卷内结构与关系：删除
- 向知识系统发送来源状态更新

### 13.9 页面级统一规则

- 当前阶段页面只对应当前阶段
- 阶段一结束，页面立即消失
- 新阶段生成新页面
- Portal 不保留旧页面列表

## 14. 动态语言

所有会动的东西都按 `Apple` 的动态语言处理。

动作语法固定为：
- 点开：进入对象下一层
- 返回：回到上一层
- 下拉：刷新当前层现实
- 长按 / 拖起：进入对象操作态
- 放下：完成对象操作

## 15. 视觉自检

每次做前台实现时，必须做以下自检：

1. 本地打开真实页面
2. 截取关键状态图
3. 像人一样检查：
   - 字体大小关系
   - 文案位置
   - 对齐
   - 留白
   - 页面呼吸感
   - 主次层级
   - `Claude` 壳是否被破坏
   - `Apple` 动态是否退化成普通 web 动画
