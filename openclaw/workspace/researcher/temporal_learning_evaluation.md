## Recommended Learning Path

### Phase 1: Foundation (1–2 weeks)
1. **High‑level overview**: Watch “Temporal in 7 Minutes” on YouTube.
2. **Official documentation**: Read “Understanding Temporal” and “Quickstarts” sections.
3. **First course**: Enroll in **Temporal 101** on Learn Temporal for your preferred programming language (Go, Java, .NET, Python, Ruby, TypeScript).
4. **Hands‑on**: Complete the course exercises; run the “Hello World” workflow locally.

### Phase 2: Hands‑on Practice (2–3 weeks)
1. **Deepen knowledge**: Take **Temporal 102** course (testing, debugging, deployment).
2. **Code examples**: Clone the official GitHub repository and explore sample applications.
3. **Build something**: Create a simple workflow that integrates an external service (e.g., call a public API).
4. **Explore the CLI**: Use the Temporal CLI to inspect workflow executions.

### Phase 3: Advanced Topics (3–4 weeks)
1. **Specialized courses**: Choose from “Error Handling Strategy”, “Versioning Workflows”, “Securing Application Data”, etc., based on your needs.
2. **Official blog**: Read case‑study articles (e.g., “Running GitHub Actions through Temporal”) to see how Temporal solves real problems.
3. **Join the community**: Sign up for the Temporal Slack workspace and attend a virtual workshop or meetup.

### Phase 4: Production Readiness (ongoing)
1. **Deployment guides**: Study the “Deploy to production” documentation.
2. **Best practices**: Review the “Best practices” section.
3. **Temporal Cloud**: If considering managed service, take the “Introduction to Temporal Cloud” course.
4. **Stay updated**: Subscribe to the Temporal newsletter and monitor the blog for new features.

## Key Takeaways
- **Highest‑credibility resources**: Official documentation and Learn Temporal platform.
- **Supplement with**: GitHub examples, official blog, and YouTube channel.
- **Use third‑party tutorials cautiously**: Verify information against official sources.
- **Community forums** (Reddit, Slack) are valuable for troubleshooting but not for foundational learning.

## Next Steps
1. Begin with Phase 1, using the official documentation and Temporal 101 course.
2. Adapt the learning path based on your specific programming language and use case.
3. Regularly check for updated courses and documentation as Temporal evolves.
