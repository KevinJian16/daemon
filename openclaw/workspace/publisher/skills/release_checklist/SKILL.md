# Release Checklist

## 适用场景
发布前执行预检清单，确认所有发布条件满足。

## 输入
待发布的版本号 + 目标分支/tag。

## 执行步骤
1. 确认所有关联 issue/PR 已关闭或合并
2. 确认 CI 测试全部通过
3. 确认 CHANGELOG 已更新
4. 确认版本号一致（pyproject.toml / package.json 等）
5. 确认无未提交的变更
6. 生成检查报告

## 质量标准
- 任一必选项未通过则阻止发布
- 报告必须包含每项的检查结果和证据

## 常见失败模式
- 版本号在多处不一致
- 遗漏检查依赖版本锁定文件

## 输出格式
```
Release v{版本号} 预检: READY / BLOCKED
[PASS/FAIL] 检查项 | 详情
```
